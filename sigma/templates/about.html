API Quick start guide

List all signatures
curl -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" https://hostname:port/api/v1/sig/

List all types
curl -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" https://hostname:port/api/v1/type/

List one signature by sig_id
curl -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" https://hostname:port/api/v1/sig/1/

List one type by type id
curl -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" https://hostname:port/api/v1/type/1/

Search types (exact match) by name, id, (partial match) by name_like, comment
curl -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" https://hostname:port/api/v1/type/?name=md5

Search signature (exact match) by type, type_id, text, id, type, (partial match) by type_like, text_like, comment
curl -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" https://hostname:port/api/v1/sig/?type=snort

Create new type
curl -X POST -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" http://127.0.0.1:8000/api/v1/type/ -d '{"name":"ja3", "comment": "ssl fingerprint md5hash"}'

Create new signature (must know the type_id)
curl -X POST -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" http://127.0.0.1:8000/api/v1/sig/ -d '{"text":"baddomain.com", "type":"bad_domain"}'

Delete type by type id (successful delete returns a 204 No Content)
This will fail if the type has signatures related to it. Signatures related to a type must be deleted first before the type will delete successfully
curl -X DELETE -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" http://127.0.0.1:8000/api/v1/type/5/

Delete signature by sig id (successful delete returns a 204 No Content)
curl -X DELETE -s -k -H "Authorization: Token your-api-key" -H "content-type: application/json" http://127.0.0.1:8000/api/v1/sig/7/
